0 · The failure we start from — naive whole-video inference

One call over 2:07 of video: temporal attention dilutes, so events late in the video vanish and directions flip. Measured: precision 0.5, recall 0.25 (Gemini flash, live) — it found 1 of 4 changes and got its direction wrong.

1 · Part 1 — overlapping windows through the same black box

No re-encoding: the notebook's own gemini_inference takes startOffset/endOffset, so chunking = one upload + N windowed calls. 20s windows with 5s overlap — the longest real maneuver is 7s, so every event fits whole in at least one window and nothing is cut at a boundary.

2 · Part 2 — adversarial judge per candidate (+ a zero-API confirmer)

Each candidate is re-inferred on its ±3s snippet at fps=5 with a prompt whose job is to REJECT: curvature, straddling, other cars' lane changes. 3 votes, majority wins; the judge saw more frames, so its direction overrides the detector's. In parallel, a pixel-only confirmer (lane-marking sweep via 1-D cross-correlation, zero model calls) boosts confidence and breaks direction ties — measured 0/7 false positives, direction 3/3, so it confirms but never vetoes.

3 · The decisions that matter (and why)

Chunk 20s / overlap 5s, derived not guessedLongest labeled maneuver is 7s → overlap ≥ 7s risk window; chunk ≫ maneuver keeps temporal attention. Their worked example (10s, no overlap) is one parameter setting: make_windows(60,10,0).
Defensive timestamp remapWindow-relative is the contract, but VLMs sometimes answer in global time — remap prefers relative, falls back to global when relative overflows the window, clamps nonsense. Every branch unit-tested.
Merging is code, not another LLM callInterval union over same-direction events (≤2s gap) dedupes double-seen events AND folds "double lane change" into one event — deterministic, free, testable offline.
Recall first, then precisionDetection samples each window 3× and unions (a borderline maneuver may appear on one sample only); the judge restores precision downstream. Measured live: single-sample detection flickered 1↔0 candidates on the same window.
Model-agnostic by seamThe whole pipeline takes infer_fn as a parameter. Same code ran live on Gemini (native video) and Claude (ffmpeg frame sampling) — and on deterministic fakes in 30 offline tests.
Quota is a design inputestimate_calls() does the math before any run: a full pass needs ~25 calls; the free Gemini tier allows 20/day. Two real runs died learning this — now it's a function with a test.
Every stage is scored, never vibedGreedy IoU≥0.3 matching vs the 4 ground-truth windows prints precision/recall/F1 for naive → chunked → validated. Improvements are claims with numbers attached.
Honest failure reportingThe pixel confirmer's first design (2-D phase correlation) measured NOT separable and was rebuilt; its final role is stated from measurements (confirm-only), not hope. Free-tier partial runs are banked and labeled as partial.
